# Markdown Notes - 产品需求文档 (PRD)

## 1. 产品概述

**项目名称**: Markdown Notes
**项目类型**: Web 笔记应用
**核心价值**: 简洁、高效的 Markdown 笔记管理，支持实时预览和语法高亮
**目标用户**: 需要创建和管理 Markdown 笔记的技术人员、写作爱好者

## 2. 用户故事

### 2.1 核心用户故事

| 用户故事 | 描述 | 优先级 |
|---------|------|--------|
| 创建笔记 | 用户可以创建新笔记并立即开始编辑 | P0 |
| 编辑笔记 | 用户可以编辑已有笔记的标题和内容 | P0 |
| 删除笔记 | 用户可以删除不需要的笔记（带确认） | P0 |
| 搜索笔记 | 用户可以通过标题快速找到笔记 | P0 |
| 实时预览 | 用户在编辑时可以同时看到渲染后的效果 | P0 |
| 代码高亮 | 用户书写的代码块有专业的语法高亮 | P0 |
| 数据持久化 | 用户的笔记自动保存在浏览器中 | P0 |

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 笔记列表
- 左侧栏顶部显示所有笔记列表
- 每个列表项显示笔记标题
- 当前选中的笔记高亮显示
- 每个列表项右侧有删除按钮

#### 3.1.2 搜索过滤
- 搜索框位于笔记列表上方
- placeholder: "搜索笔记..."
- 实时按标题过滤，无需回车

#### 3.1.3 新建笔记
- 按钮文本: "+ 新建笔记"
- 点击后创建空白笔记
- 自动聚焦到编辑区域

#### 3.1.4 编辑器
- textarea 受控组件
- 显示当前选中笔记的 Markdown 内容
- 编辑时实时更新状态

#### 3.1.5 实时预览
- 右侧区域使用 react-markdown 渲染
- 支持 GFM (表格、删除线、任务列表等)
- 代码块使用 react-syntax-highlighter (oneDark 主题)

#### 3.1.6 删除笔记
- 点击删除按钮显示 window.confirm() 确认框
- 确认后删除笔记

#### 3.1.7 数据持久化
- 所有笔记保存到 LocalStorage
- Key: `markdown-notes`
- 应用启动时自动加载

## 5. 技术需求

### 5.1 技术栈

| 类别 | 技术选型 | 版本要求 |
|------|---------|---------|
| 前端框架 | React + TypeScript | React 18+ |
| 构建工具 | Vite | 最新版 |
| 样式方案 | Tailwind CSS | 4.x |
| Markdown 渲染 | react-markdown | 9.x |
| GFM 插件 | remark-gfm | 4.x |
| 代码高亮 | react-syntax-highlighter | 15.x |
| 图标 | lucide-react | 最新版 |

### 5.2 数据模型

```typescript
interface Note {
  id: string;          // crypto.randomUUID() 生成
  title: string;       // 笔记标题
  content: string;     // Markdown 内容
  createdAt: number;   // 创建时间戳
  updatedAt: number;   // 更新时间戳
}
```

## 6. 验收标准

### 6.1 功能验收

| 功能点 | 验收条件 |
|--------|---------|
| 新建笔记 | 点击按钮后列表出现新笔记，编辑区聚焦 |
| 编辑笔记 | 修改内容后预览实时更新 |
| 删除笔记 | 确认后笔记从列表移除 |
| 搜索笔记 | 输入关键词后列表实时过滤 |
| 数据持久化 | 刷新页面后笔记数据保留 |
| 代码高亮 | 代码块有 oneDark 主题颜色 |

### 6.2 视觉验收

| 检查点 | 标准 |
|--------|------|
| 布局 | 左右分栏比例约 40:60 |
| 响应式 | 宽度 <1024px 时正常显示 |
| 交互反馈 | hover、focus 状态可见 |
| 代码块 | 有 padding、滚动条、深色背景 |

## 7. 非功能性需求

### 7.1 兼容性
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

### 7.2 可访问性
