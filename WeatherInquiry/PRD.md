# 天气查询系统 — 产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称
天气查询（WeatherInquiry）

### 1.2 产品定位
一款基于 React 19 和 TypeScript 构建的轻量级单页应用（SPA），为用户提供实时天气查询、空气质量监测、天气预报、天气预警、多城市对比和城市收藏等功能。

### 1.3 目标用户
- 需要快速查询城市天气的普通用户
- 需要对比多个城市天气数据的出行规划者
- 关注空气质量和天气预警的敏感人群

### 1.4 核心价值
- **即时查询**：输入城市名称或自动定位，即时获取天气数据
- **信息全面**：涵盖温度、湿度、风速、能见度、空气质量等多项指标
- **智能预警**：根据天气条件自动检测并推送预警信息
- **多城对比**：同时对比最多 6 个城市的天气数据
- **收藏管理**：常用城市一键收藏，快速查看

## 2. 功能需求

### 2.1 天气查询功能

| 需求编号 | 功能描述 | 优先级 |
|---------|---------|--------|
| F-001 | 支持通过城市名称搜索天气（支持中文、英文、拼音） | P0 |
| F-002 | 支持通过浏览器地理定位获取当前位置天气 | P0 |
| F-003 | 显示实时温度、体感温度、天气描述、湿度、风速、能见度 | P0 |
| F-004 | 显示天气图标，并根据天气类型提供不同的动画效果 | P1 |
| F-005 | 搜索时显示加载动画，查询完成或失败后自动隐藏 | P0 |
| F-006 | 查询失败时显示友好的错误提示，5 秒后自动消失 | P1 |

### 2.2 空气质量功能

| 需求编号 | 功能描述 | 优先级 |
|---------|---------|--------|
| F-010 | 显示 AQI 指数、等级标签（优/良/轻度/中度/重度）及描述 | P1 |
| F-011 | 显示 PM2.5、PM10、臭氧（O₃）、二氧化氮（NO₂）等关键指标 | P1 |
| F-012 | AQI 数据获取失败时不影响主天气数据显示 | P2 |

### 2.3 天气预报功能

| 需求编号 | 功能描述 | 优先级 |
|---------|---------|--------|
| F-020 | 显示未来 5 天的天气预报（不包含当天） | P1 |
| F-021 | 每天显示日期、星期、天气描述、最高/最低温度、天气图标 | P1 |
| F-022 | 自动将 3 小时预报数据按天聚合 | P1 |

### 2.4 天气预警功能

| 需求编号 | 功能描述 | 优先级 |
|---------|---------|--------|
| F-030 | 自动检测高温（≥35℃）、严寒（≤-10℃）、低温（≤0℃）等温度异常 | P1 |
| F-031 | 自动检测大风（风速 ≥10m/s） | P1 |
| F-032 | 自动检测闷热（湿度 ≥90% 且温度 ≥25℃） | P2 |
| F-033 | 自动检测大雾/雾霾（能见度低于 5000m） | P1 |
| F-034 | 自动检测雷暴、暴雨、大雪等极端天气 | P1 |
| F-035 | 预警按严重程度分级（红/橙/黄/蓝），以不同颜色标识 | P1 |
| F-036 | 无预警时不显示预警模块 | P2 |

### 2.5 多城市对比功能

| F-042 | 最多支持 6 个城市同时对比 | P1 |

| F-044 | 点击城市名可跳转查看该城市的详细天气信息 | P2 |
| F-045 | 支持从对比列表中移除城市 | P1 |
| F-046 | 防止重复添加同一城市 | P2 |

### 2.6 城市收藏功能

| 需求编号 | 功能描述 | 优先级 |
|---------|---------|--------|
| F-050 | 支持收藏/取消收藏当前查看的城市 | P1 |

## 3. 非功能需求

### 3.1 性能要求
| 指标 | 要求 |
|------|------|
| 首屏加载时间 | 小于 2 秒 |
| API 请求超时 | 25 秒 |
| 自动重试次数 | 最多 2 次（共 3 次请求） |
| 重试间隔 | 2 秒 |
| 错误提示自动消失 | 5 秒 |

### 3.2 兼容性要求
- 现代浏览器（Chrome、Firefox、Edge、Safari 最新版本）
- 移动端自适应（支持 480px 以下屏幕）
- 支持 HTTPS 环境（浏览器地理定位需要）

### 3.3 安全性要求
- 开发环境 API 密钥由代理服务器注入，不在浏览器请求中暴露
- 生产环境 API 密钥通过环境变量注入，编译后打包到代码中
- 用户收藏数据仅存储在本地浏览器 localStorage 中

### 3.4 网络容错要求
- 支持超时自动中断（AbortController）
- 网络波动时自动重试
- API 错误（401/404/429）提供明确的用户提示

## 4. 技术架构

### 4.1 技术栈
| 层级 | 技术 | 版本 |
|------|------|------|
| 前端框架 | React | 19.2.5 |
| 类型系统 | TypeScript | 6.0.2 |
| 构建工具 | Vite | 8.0.10 |
| 代码规范 | ESLint + typescript-eslint | 10.2.1 |
| 网络请求 | 原生 Fetch API | 浏览器内置 |
| 第三方 API | OpenWeatherMap | v2.5 |

### 4.2 架构设计
```
src/
├── api/              # API 调用层（weather.ts）
│   ├── 类型定义      # 接口类型
│   ├── 请求工具      # 重试、超时、错误处理
│   └── 数据转换      # URL 构建、AQI 等级、预警检测
├── components/       # UI 组件层
│   ├── App.tsx       # 主应用（状态管理、路由协调）
│   ├── SearchBar     # 搜索栏
│   ├── WeatherCard   # 天气卡片（当前天气 + AQI + 预报）
│   ├── WeatherAlert  # 天气预警
│   ├── CityCompare   # 多城市对比
│   ├── Favorites     # 城市收藏
│   └── Spinner       # 加载动画
├── App.css           # 全局样式（含动画）
└── index.css         # 基础 CSS 变量
```

### 4.3 开发代理配置
- 开发模式：Vite 代理 `/api/openweather` → `https://api.openweathermap.org`
- 路径重写：自动去除前缀并注入 `appid` 参数
- 生产模式：前端直连 OpenWeatherMap API

## 5. 接口定义

### 5.1 使用的 OpenWeatherMap 接口

| 接口 | 用途 | 参数 |
|------|------|------|
| `/data/2.5/weather` | 获取当前天气 | q（城市名）或 lat/lon（经纬度） |
| `/data/2.5/forecast` | 获取 5 天预报 | q（城市名）或 lat/lon（经纬度） |
| `/data/2.5/air_pollution` | 获取空气质量 | lat/lon（经纬度） |

### 5.2 通用请求参数
| 参数 | 值 | 说明 |
|------|----|------|
| units | metric | 使用摄氏度 |
| lang | zh_cn | 中文返回 |
| appid | API Key | 开发环境由代理注入 |

## 6. 状态管理

应用使用 React 内置的 `useState` 和 `useCallback` 进行状态管理，未引入外部状态管理库。

| 状态 | 类型 | 作用域 |
|------|------|--------|
| currentWeather | CurrentWeather \| null | 当前天气数据 |
| forecast | ForecastResponse \| null | 预报数据 |
| aqi | AqiData \| null | 空气质量数据 |
| loading | boolean | 全局加载状态 |
| error | string \| null | 错误信息 |
| locating | boolean | 定位加载状态 |
| showCompare | boolean | 是否显示对比面板 |
