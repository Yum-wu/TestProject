一、中文版
1. 产品概述
DeepSeek 是深度求索（DeepSeek）公司开发的新一代开源大语言模型系列。公司秉持“技术开源、生态共建”的发展战略，致力于将高性能 AI 能力普惠化，向全球开发者和用户开放前沿大语言模型技术。

2026 年 4 月 24 日，DeepSeek 正式发布 DeepSeek-V4 系列模型预览版，并同步向全球开源。该系列模型突破性地支持 100 万 token（约 75 万字）超长上下文，在智能体协同能力、世界知识储备及复杂逻辑推理性能上，均实现国内与开源领域的全面领先。DeepSeek 始终坚守开源与普惠路线。普通用户可通过官网或官方 App 完全免费使用，不限次数；API 面向开发者按量计费，定价极具竞争力，Pro 版本更于 2026 年 5 月 22 日宣布永久降价至原价的 25%，进一步降低企业级 AI 应用门槛。

2. 核心特性
2.1 百万 Token 超长上下文
DeepSeek-V4 全系标配 100 万 token（约 75 万字）的超长上下文窗口，无需额外付费或特殊配置。这意味着模型可以一次性读完相当于《三体》三部曲体量的内容，并保持精准的理解与记忆。

相比之下，传统开源模型的上下文窗口多局限于几十 K 到几百 K，面对整本书籍、完整代码仓库、大型合同与企业知识库时易出现信息截断、逻辑断裂。DeepSeek-V4 通过自研 DSA 稀疏注意力机制与全新混合注意力架构，在 token 维度压缩计算量，大幅降低百万上下文场景下的显存与计算开销，使超长文本的全局理解与精准问答成为现实。

2.2 领先的智能体（Agent）能力
V4-Pro 的智能体能力达到开源模型最佳水平，在 Agentic Coding 评测中登顶开源模型榜首，已成为 DeepSeek 内部员工的主力编程模型，实际体验优于 Claude Sonnet 4.5，交付质量接近 Claude Opus 4.6 非思考模式。

模型已针对 Claude Code、OpenClaw、OpenCode、CodeBuddy 等主流智能体框架完成专项适配和优化，在自动化代码生成、结构化文档输出（如 PPT/报告生成）等任务中表现显著提升。模型支持复杂任务拆解、多步骤推理与工具调用，可高效完成代码生成、文档处理、数据分析、自动化工作流搭建等智能体任务。

2.3 世界知识与推理性能
在世界知识、数学、STEM 竞赛、代码生成等专项评测中，V4-Pro 刷新开源模型纪录，部分指标接近甚至超越部分顶级闭源模型。模型强化了专业领域知识储备，兼顾通用性与专业性，既能满足日常对话、文案创作，也可适配法律、金融、工程、科研等专业场景的精细化需求。

2.4 双版本设计
DeepSeek-V4 分为 Pro 和 Flash 两个版本，精准适配不同应用场景：

V4-Pro（高性能旗舰版） ：总参数约 1.6 万亿，激活参数约 490 亿，专注于 Agentic Coding、数学推理、世界知识等复杂场景，适合高要求的智能体任务、长文档分析和专业代码生成。

V4-Flash（高效经济版） ：总参数约 2840 亿，激活参数约 130 亿，推理能力接近 Pro 版，但世界知识储备略有缩减，成本更低、响应更快，适合日常对话、轻量知识库和大规模 API 调用等场景。

2.5 思考模式与双模式推理
V4-Pro 与 V4-Flash 均同时支持非思考模式与思考模式。思考模式支持 reasoning_effort 参数设置思考强度（high / max），对于复杂的智能体场景建议使用思考模式并设置为 max。

非思考模式（Non-Think） ：响应速度最快，适合路由、分类等轻量级任务。

思考模式（Think High / Think Max） ：深度推理，适合代码生成、复杂分析和智能体任务，在准确性要求最高的场景推荐使用 Think Max。

2.6 开源与生态兼容
DeepSeek-V4 延续“技术开源、生态共建”战略，预览版权重已在 Hugging Face 开源社区发布，支持本地部署、私有化部署、二次微调与定制化开发。模型权重采用 MIT 许可证发布，任何人可自由下载与修改。

API 层面兼容 OpenAI Chat Completions 与 Anthropic Messages 两套主流接口格式，开发者只需修改一行配置代码即可快速迁移和集成，大幅降低技术切换成本。

2.7 联网搜索（Web Search）
DeepSeek 支持联网搜索功能。在官网或 App 中，用户可在对话输入框附近手动开启联网搜索开关（图标为地球或信号波纹）。该功能默认关闭，每次对话独立控制，开启后模型可获取实时信息，拓展知识范围和回答准确性。

2.8 文件上传支持
在官网或 App 对话界面，用户可上传多种格式文件，包括 PDF、Word（.docx）、Excel（.xlsx）、PowerPoint、TXT 以及 PNG/JPG 等图片格式。上传后可直接提问，模型将自动进行 OCR 文字识别（针对图片文件）或内容提取与理解。

3.2 架构创新：混合注意力机制
DeepSeek-V4 在底层架构上实现重大创新，采用全新的混合注意力架构，融合了压缩稀疏注意力（CSA）与严重压缩注意力（HCA），并替代传统残差连接使用了流形约束超连接（mHC）。

相比 DeepSeek-V3.2，这些创新可将每个 token 推理所需的 FLOPs 减少 73%，并将 KV 缓存的内存负载降低 90%。KV 缓存是长上下文推理的主要瓶颈——智能体可以携带系统指令、工具输出、检索到的上下文、代码、日志、内存和多步骤推理痕迹，随着上下文窗口增长，注意力机制和 KV 缓存压力剧增。DeepSeek-V4 的架构创新从根本上改变了推理经济学的动态，使长上下文智能体工作流在实际部署中变得可行。

4. 使用方式与指南
DeepSeek-V4 提供多种使用途径：

4.1 网页版（Web Chat）
入口地址：https://chat.deepseek.com

特点：即开即用，零门槛体验，适合日常问答、文档撰写或快速测试模型功能。支持文件上传、联网搜索和完整 1M 上下文。

使用方法：访问上述地址，注册/登录账号后，在对话输入框中直接输入问题即可获得响应。

模型选择：登录后可在模型列表中选择 V4 版本，支持在“专家模式”（V4-Pro）和“快速模式”（V4-Flash）之间切换。默认使用 V4-Pro 模型。

注册方式：支持手机号验证码、微信快捷登录、邮箱注册三种方式。

网页端免费使用，不限次数。

4.2 官方 App
获取方式：在手机应用商店（iOS App Store / 安卓应用市场）搜索“DeepSeek”，认准开发商为“深度求索”。

功能：与网页端功能完全同步，适合在通勤、出差等移动场景下使用。

使用限制：灰度测试用户需将 App 更新至 1.7.4 或更高版本，才能体验 V4 版本的百万上下文功能。

App 端同样完全免费使用，不限次数。

4.3 API 接入
适用于需要将 DeepSeek-V4 集成到自有应用、服务或工作流中的开发者。

API Base URL（OpenAI 格式） ：https://api.deepseek.com

API Base URL（Anthropic 格式） ：https://api.deepseek.com/anthropic

模型名称参数：

deepseek-v4-pro（Pro 旗舰版，性能更强，适合复杂任务）

deepseek-v4-flash（Flash 高效版，响应更快、成本更低，适合日常调用）

API 兼容性：同时兼容 OpenAI Chat Completions 和 Anthropic Messages 两套主流接口标准。

4.4 API 首次调用（Python 示例）
python
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get('DEEPSEEK_API_KEY'),
    base_url="https://api.deepseek.com"
)

response = client.chat.completions.create(
    model="deepseek-v4-pro",
    messages=[
        {"role": "system", "content": "You are a helpful assistant"},
        {"role": "user", "content": "Hello"}
    ],
    stream=False,
    reasoning_effort="high",
    extra_body={"thinking": {"type": "enabled"}}
)

print(response.choices[0].message.content)

4.8 提示（Prompt）优化建议
根据官方推荐：

Temperature 设置：代码生成和规范写作建议设为 0.2；创意任务和措辞探索建议设为 0.5；创意写作和头脑风暴可设为 0.7+。

提示风格：建议使用直接、明确的提示语，跳过客套的开场白。例如：直接要求“Write a Python function to sort a list of dictionaries by a key”，而不是“Can you help me with...”。

长对话注意事项：如果模型的回复开始偏离主题或变得模糊，建议开启新对话线程而非在原对话中持续修正。V4 在新对话中的表现优于长累积上下文。
